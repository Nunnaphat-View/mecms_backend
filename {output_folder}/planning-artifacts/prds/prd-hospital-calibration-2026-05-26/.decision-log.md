# บันทึกการตัดสินใจการออกแบบระบบ (System Design Decision Log)

ไฟล์นี้เก็บประวัติการตัดสินใจ การปรับเปลี่ยนเงื่อนไข และข้อยอมรับร่วมกันของระบบสอบเทียบและบำรุงรักษาเครื่องมือแพทย์ระดับโรงพยาบาล

---

## ประวัติการตัดสินใจ (Decision History)

### [DEC-001] การตั้งค่าเริ่มต้นและการเลือกแนวทางการออกแบบ
- **วันที่:** 2026-05-26
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - กำหนดให้ระบบรองรับการสแกน **QR Code** ในการสืบค้นและระบุตัวตนเครื่องมือแพทย์ในแต่ละขั้นตอน
  - กำหนดให้ระบบปฏิบัติตาม **มาตรฐานการสอบเทียบและบำรุงรักษาทางการแพทย์** ในระดับความเข้มงวดสูงของระดับโรงพยาบาล (Hospital-level rigour)
  - เลือกทำงานในโหมด **Coaching path** (ดำเนินกระบวนการแบบร่วมมือและเรียนรู้ร่วมกัน)

### [DEC-002] การเลือกจุดเริ่มต้นเอกสาร PRD
- **วันที่:** 2026-05-26
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - เลือกโครงสร้างการระดมสมองและจุดเริ่มต้นการสร้าง PRD ด้วยแนวทาง **Vision + Features** (เริ่มจากวิสัยทัศน์ของระบบตามด้วยคุณสมบัติหลัก)

### [DEC-003] การกำหนดวิสัยทัศน์ ตัวชี้วัด และข้อจำกัดระบบ
- **วันที่:** 2026-05-27
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - **วิสัยทัศน์หลัก:** ต้องการทดแทนระบบการจดบันทึกด้วยกระดาษ (Paperless) เพื่อลดความผิดพลาดจากคนทำงาน (Human error) และเพิ่มความแม่นยำความน่าเชื่อถือของเครื่องมือแพทย์เพื่อความปลอดภัยสูงสุดของคนไข้
  - **ตัวชี้วัดความสำเร็จ:** สามารถออกใบรายงานผลการสอบเทียบได้ทันทีหลังจากหัวหน้าอนุมัติ (Instant report generation) และจัดเก็บใบรายงานผลอย่างเป็นระบบสืบค้นได้ง่าย
  - **ข้อจำกัดและความปลอดภัย:** ประวัติการซ่อม บำรุงรักษา และการสอบเทียบทั้งหมด ต้องมีความปลอดภัยสูงและไม่สามารถแก้ไขข้อมูลย้อนหลังได้ (Immutable log) เพื่อรักษาความถูกต้องของข้อมูลในการสืบทวน

### [DEC-004] การกำหนดฟีเจอร์และฟังก์ชันการทำงานหลัก (Core Features)
- **วันที่:** 2026-05-27
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - ตกลงใช้ฟีเจอร์หลักทั้ง 6 โมดูล ได้แก่:
    1. ระบบสแกน QR Code เพื่อระบุตัวตนและดึงข้อมูลเครื่องมือ
    2. ระบบเช็คลิสต์บำรุงรักษาเชิงป้องกัน (PM)
    3. ระบบคำนวณและประเมินผลสอบเทียบ (Calibration)
    4. ระบบการอนุมัติตรวจสอบ 2 ระดับ (หัวหน้าแผนก)
    5. ระบบพิจารณาจัดการเครื่องชำรุด (ผอ. พิจารณา ซ่อม/จำหน่าย)
    6. ระบบออกรายงาน PDF และเก็บประวัติถาวรแบบแก้ไขไม่ได้ (Immutable log)

### [DEC-005] การเห็นชอบบทบาทและเส้นทางการทำงานของกลุ่มผู้ใช้
- **วันที่:** 2026-05-27
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - เห็นชอบและยอมรับบทบาทสมมติ (Protagonists) และเส้นทางการทำงาน (User Journeys) ของช่างสมชาย, หัวหน้าศิริพงษ์, และ ผอ.วิชัย ตามร่างที่ระบุใน PRD ไม่มีข้อน่ากังวลหรือการปรับปรุงเพิ่มเติมในขณะนี้

### [DEC-006] การปิดเล่มและสรุปสถานะ PRD
- **วันที่:** 2026-05-27
- **สถานะ:** ยืนยันแล้ว
- **รายละเอียด:**
  - ทำการปิดเล่ม PRD และปรับสถานะเป็น `status: final`
  - ทำการออกเอกสารสรุปความต้องการใช้งาน (PRD) และรายละเอียดเทคนิคเพิ่มเติม (Addendum) พร้อมส่งมอบเพื่อเตรียมดำเนินการออกแบบ UI (bmad-ux) หรือวางแผนงานสถาปัตยกรรม (bmad-create-architecture) ต่อไป
